# Safari Business Seed Data Documentation

This document provides a detailed overview of the "Demo Safari Company" dataset generated by the `seed_business` management command. This data is designed to simulate a 12-24 month operational state for the Successful Safaris platform.

## 🛠️ Seeding Commands

### Full Business Reset
To wipe existing business data and repopulate with the demo dataset:
```bash
python manage.py seed_business --clear
```
*Note: This command requires explicit `YES` confirmation to prevent accidental wipes.*

### Incremental Seeding
To add missing entities without clearing existing ones:
```bash
python manage.py seed_business
```

## 🏗️ Seeding Architecture

The seeding process is divided into four distinct phases to ensure data integrity and respect foreign key dependencies.

### Phase 1: Core Identity Layer
- **User Roles:** Seeds 9 standard industry roles (Visitor, Customer, Travel Consultant, Operations Officer, Regional Manager, Finance Officer, Content Manager, Sub Admin, Super Admin).
- **Users:** Generates 30 realistic demo customers with unique email addresses and passwords.
- **Profiles:** Creates extended `UserProfile` records for all customers, including nationality, passport data (faked), and loyalty tiers (Bronze to Platinum).
- **Avatars:** Downloads unique human avatars for every customer profile.

### Phase 2: Product Catalog
- **Destinations:** Seeds the "Eight Jewels" of Africa (Kenya, Tanzania, Rwanda, Uganda, Botswana, Namibia, South Africa, Zambia) with country-specific hero images.
- **Safari Locations:** Populates 26 national parks and reserves with unique landscape photography.
- **Accommodations:** Generates 52 luxury lodges and tented camps with distinct architectural visuals and nightly rates ranging from $400 to $2,500.
- **Packages:** Creates 30 signature safari packages. Each package includes:
    - Unique action-oriented featured image.
    - Full day-by-day itinerary with localized descriptions.
    - Inclusion/Exclusion lists.
- **Shop Inventory:** Seeds 40 high-end products across 7 categories (Clothing, Optics, Gear, etc.) with unique, category-aware imagery.

### Phase 3: Transactional History
- **Bookings:** Generates 20 historical and upcoming bookings across different statuses (Draft, Pending, Confirmed, Completed, Cancelled).
- **Travelers:** Attaches 1-4 traveler records to each booking with full KYC data (Name, DOB, Nationality).
- **Payments:** Generates transaction records (Card, M-Pesa, Bank Transfer) linked to bookings to simulate revenue flow.

### Phase 4: Content & Engagement
- **Safari Journal:** Seeds 25+ blog posts across 5 categories with unique wildlife and landscape photography.
- **Customer Reviews:** Generates authentic-looking 4-5 star reviews for featured packages.
- **CRM Inquiries:** Seeds 30 inquiry records to populate the staff support dashboard.

## 🖼️ Visual Asset Management

- **Unique IDs:** The seeder uses a curated library of 100+ unique Unsplash Photo IDs to prevent visual repetition.
- **Local Storage:** All images are downloaded during the first seeding run and stored in the `/media/` directory.
- **Caching:** An internal download cache is used to speed up subsequent runs and avoid redundant network requests.
